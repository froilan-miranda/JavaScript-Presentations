History of Javascript

  • Do not confuse JavaScript with Java
  • JavaScript was created in 10 days in May 1995 by Brendan Eich
  • Brendan Eich at the time worked at Netscape and now of Mozilla

History of Javascript

  • The original name was Mocha, chosen from the founder of Netscape
  • September of 1995 the name was changed to LiveScript
  • December of the same year, it received a trademark license from Sun to use JavaScript
  • It was somewhat of a marketing move, because of the popularity of Java at the time
  • Do not confuse JavaScript with Java

History of Javascript

  • 1996 - 1997 JavaScript was taken to ECMA to carve out a standard specification
  • European Computer Manufacturers Association (ECMA)
  • This would allow other browser vendors to implement based on the work done at Netscape
  • ECMAScript is the name of the official standard with JavaScript being the most well known implementation
  • ActionScript 3 is another well-known implementation of ECMAScript

History of Javascript

  • The standards process continued in cycles
  • 1998 - ECMAScript 2
  • 1999 - ECMAScript 3 (baseline of modern JavaScript)
  • 2000 - The "JS2" or "original ES4" work led by Waldemar Horwat (then of Netscape, now at Google) started
  • At first, Microsoft seemed to participate and even implemented some of the proposals in their JScript.net language.

History of Javascript

  • It became clear though that Microsoft had no intention of cooperating or implementing proper JS in IE
  • Even though they had no competing proposal and they had a partial (and diverged at this point) implementation on the .NET server side.
  • By 2003 the JS2/original-ES4 work was mothballed.

History of Javascript

  • The next major event was in 2005, with two major happenings in JavaScript’s history
  • First, Brendan Eich and Mozilla rejoined Ecma as a not-for-profit member and work started on E4X, ECMA-357, which came from ex-Microsoft employees at BEA (originally acquired as Crossgain).
  • This led to working jointly with Macromedia, who were implementing E4X in ActionScript 3(ActionScript 3 was a fork of Waldemar's JS2/original-ES4 work).

History of Javascript

  • So, along with Macromedia (later acquired by Adobe), work restarted on ECMAScript 4 with the goal of standardizing what was in AS3 and implementing it in SpiderMonkey.
  • To this end, Adobe released the "AVM2", code named Tamarin, as an open source project. But Tamarin and AS3 were too different from web JavaScript to converge, as was realized by the parties in 2007 and 2008.

History of Javascript

  • Alas, there was still turmoil between the various players; Doug Crockford — then at Yahoo! — joined forces with Microsoft in 2007 to oppose ECMAScript 4, which led to the ECMAScript 3.1 effort.
While all of this was happening the open source and developer communities set to work to revolutionize what could be done with JavaScript.
This community effort was sparked in 2005 when Jesse James Garrett released a white paper in which he coined the term Ajax, and described a set of technologies, of which JavaScript was the backbone, used to create web applications where data can be loaded in the background, avoiding the need for full page reloads and resulting in more dynamic applications.
This resulted in a renaissance period of JavaScript usage spearheaded by open source libraries and the communities that formed around them, with libraries such as Prototype, jQuery, Dojo and Mootools and others being released.

In July of 2008 the disparate parties on either side came together in Oslo.
This led to the eventual agreement in early 2009 to rename ECMAScript 3.1 to ECMAScript 5 and drive the language forward using an agenda that is known as Harmony.

All of this then brings us to today, with JavaScript entering a completely new and exciting cycle of evolution, innovation and standardisation, with new developments such as the Nodejs platform, allowing us to use JavaScript on the server-side, and HTML5 APIs to control user media, open up web sockets for always-on communication, get data on geographical location and device features such as accelerometer, and more.
